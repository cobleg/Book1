2.2 Additional information

There is information available on the Internet that provides guidance on producing and publishing bookdown artefacts such as books, technical reports, academic papers etc. Here is a shortlist that I found helpful:

  • Yihui Xie (2022). bookdown: Authoring Books and Technical Documents with R Markdown, CRC Publishing.

  • R for the Rest of Us

  • Hosting bookdown in github

  • Bookdown and GitHub Pages

  • SSH Permission Denied (PublicKey) Error on GitHub Solved

  • Alison Hill (2021). Day 04: bookdown || rmarkdown + bookdown + GitHub actions + renv + Netlify

2.2.1 Summary of what I learned

My main focus has been how to:

  • Adapt the format of bookdown to produce technical reports.

  • Publish on GitHub

  • Export to Word since most clients still want a technical report in Word.

My outline here is based on exception, i.e. deviations from what was demonstrated in the shortlist presented above. This reflects gaps in my knowledge, which led to considerable frustration.
